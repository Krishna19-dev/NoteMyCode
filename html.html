HTML Programming

Comprehensive notes on HTML (HyperText Markup Language)

Introduction to HTML

HTML (HyperText Markup Language) is the standard markup language used to create and design documents on the World Wide Web. It structures web content by using a system of tags and elements that define the content and layout of web pages.

Developed by Tim Berners-Lee in 1991, HTML is the backbone of every website. It is not a programming language but a markup language used to annotate text, images, and other content for display in web browsers.

HTML works alongside CSS (Cascading Style Sheets) for styling and JavaScript for interactivity, forming the core technologies of the web. Browsers read and interpret HTML files to display websites to users.

HTML documents are plain text files saved with the .html extension and consist of elements represented by tags enclosed in angle brackets (< >), such as <p>, <h1>, <a>, etc.

HTML Document Structure

HTML documents follow a standard structure and syntax. Every HTML page starts with a DOCTYPE declaration and contains essential elements that define the document structure.

<!DOCTYPE html>
<html>
<head>
    <title>My First HTML Page</title>
</head>
<body>
    <h1>Hello, World!</h1>
    <p>This is a paragraph.</p>
</body>
</html>

The DOCTYPE declaration tells the browser which version of HTML is being used, while the html element serves as the root container for all other elements on the page.

Common HTML Elements

HTML provides a wide variety of elements for structuring content. Here are some of the most commonly used elements for creating web pages:

<!-- Headings -->
<h1>Main Heading</h1>
<h2>Sub Heading</h2>

<!-- Paragraphs and Links -->
<p>This is a paragraph with a <a href="https://example.com">link</a>.</p>

<!-- Images -->
<img src="image.jpg" alt="Description" width="300">

<!-- Lists -->
<ul>
    <li>Item One</li>
    <li>Item Two</li>
</ul>

These elements form the foundation of web content, allowing you to create structured, accessible, and meaningful web pages that browsers can properly interpret and display.

Essential HTML Tags

Understanding the core HTML tags is crucial for web development. Each tag serves a specific purpose in structuring and presenting web content.

Here are the fundamental HTML tags every developer should know:

  • <!DOCTYPE html>: Declares the document type and HTML version
  • <html>: Root element that contains all other HTML elements
  • <head>: Contains metadata, title, and links to external resources
  • <body>: Contains all visible content of the web page
  • <title>: Sets the title visible in the browser tab

These tags create the basic structure that every HTML document must have to be valid and properly rendered by web browsers.

HTML Forms and Input Elements

Forms are essential for collecting user input on web pages. They allow users to submit data like login credentials, contact information, surveys, and more.

<form action="/submit" method="post">
    <label for="name">Name:</label>
    <input type="text" id="name" name="name" required>
    
    <label for="email">Email:</label>
    <input type="email" id="email" name="email" required>
    
    <label for="message">Message:</label>
    <textarea id="message" name="message" rows="4" cols="50"></textarea>
    
    <input type="submit" value="Submit">
</form>

Forms use various input types including text, email, password, number, date, and more. The action attribute specifies where to send the form data, while the method attribute defines how to send it (GET or POST).

HTML Tables

Tables are used to display structured data in rows and columns. They're perfect for presenting information like schedules, statistics, or comparison data.

<table border="1">
    <thead>
        <tr>
            <th>Name</th>
            <th>Age</th>
            <th>City</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>Alice</td>
            <td>25</td>
            <td>New York</td>
        </tr>
        <tr>
            <td>Bob</td>
            <td>30</td>
            <td>London</td>
        </tr>
    </tbody>
</table>

Tables consist of thead (header), tbody (body), and tfoot (footer) sections. Use th for header cells and td for data cells. The border attribute adds visible borders to the table.

Semantic HTML5 Elements

Semantic HTML uses elements that clearly describe their meaning to both browsers and developers. This improves accessibility, SEO, and code maintainability.

<header>
    <nav>
        <ul>
            <li><a href="#home">Home</a></li>
            <li><a href="#about">About</a></li>
        </ul>
    </nav>
</header>

<main>
    <section>
        <article>
            <h2>Article Title</h2>
            <p>Article content goes here...</p>
        </article>
    </section>
    
    <aside>
        <h3>Related Links</h3>
        <p>Sidebar content</p>
    </aside>
</main>

<footer>
    <p>© 2025 My Website</p>
</footer>

Semantic elements like header, nav, main, section, article, aside, and footer provide clear structure and meaning to web content, making it more accessible to screen readers and search engines.

HTML Attributes and Accessibility

HTML attributes provide additional information about elements and are crucial for accessibility, functionality, and styling. They appear in the opening tag of elements.

<!-- Common attributes -->
<a href="https://example.com" target="_blank" title="Visit Example">
    External Link
</a>

<img src="image.jpg" alt="Descriptive text for screen readers" 
     width="300" height="200">

<div class="container" id="main-content" style="color: blue;">
    Content with multiple attributes
</div>

<!-- Accessibility attributes -->
<button aria-label="Close dialog" onclick="closeDialog()">×</button>
<input type="text" aria-describedby="help-text" required>
<span id="help-text">Please enter your full name</span>

Key attributes include href (links), src (images/media), alt (alternative text), class/id (styling/scripting), and ARIA attributes for enhanced accessibility. Always include alt text for images and proper labels for form elements.

HTML5 Multimedia Elements

HTML5 introduced native support for audio and video content, eliminating the need for plugins like Flash. These elements provide built-in controls and extensive customization options.

<!-- Video element -->
<video width="400" height="300" controls poster="thumbnail.jpg">
    <source src="movie.mp4" type="video/mp4">
    <source src="movie.webm" type="video/webm">
    Your browser does not support the video tag.
</video>

<!-- Audio element -->
<audio controls>
    <source src="audio.mp3" type="audio/mpeg">
    <source src="audio.ogg" type="audio/ogg">
    Your browser does not support the audio element.
</audio>

<!-- Embedded content -->
<iframe src="https://www.youtube.com/embed/VIDEO_ID" 
        width="560" height="315" frameborder="0">
</iframe>

Always provide multiple source formats for better browser compatibility. The controls attribute adds play/pause buttons, while the poster attribute sets a preview image for videos. Include fallback text for unsupported browsers.

HTML Best Practices

Following HTML best practices ensures your code is clean, accessible, maintainable, and performs well across different browsers and devices.

Essential HTML best practices include:

  • Use semantic elements: Choose elements based on meaning, not appearance
  • Validate your HTML: Use W3C validator to check for errors
  • Include alt text: Always provide alternative text for images
  • Proper nesting: Ensure elements are properly opened and closed
  • Use lowercase: Write tag names and attributes in lowercase
  • Quote attributes: Always quote attribute values
  • Optimize images: Use appropriate formats and sizes
  • Mobile-first: Include viewport meta tag for responsive design
<!-- Good HTML structure -->
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Page Title</title>
    <meta name="description" content="Page description for SEO">
</head>
<body>
    <header role="banner">
        <h1>Site Title</h1>
    </header>
    <main role="main">
        <h2>Page Heading</h2>
        <p>Well-structured content</p>
    </main>
</body>
</html>

Clean, semantic HTML forms the foundation of accessible, SEO-friendly websites that work well across all devices and assistive technologies.
