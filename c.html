C Programming

Foundation of modern programming languages

Introduction to C

C is a general-purpose, procedural computer programming language developed in the early 1970s by Dennis Ritchie at Bell Labs. It was designed to provide low-level access to memory and language constructs that map efficiently to machine instructions, all with minimal runtime support.

Despite being over five decades old, C remains one of the most widely used and influential programming languages. It has significantly influenced many other popular languages like C++, Java, JavaScript, and Python, making it an essential language to understand for any serious programmer.

C is often described as a "middle-level" language, combining elements of high-level languages with the functionality of assembly language. This unique position allows programmers to write efficient code that can interact directly with hardware while maintaining readability and portability.

The language is particularly valued for system programming, including operating systems and embedded systems, where efficiency and direct hardware manipulation are critical factors.

Basic Structure of a C Program

Every C program has a specific structure that includes preprocessor directives, global declarations, the main function, and other user-defined functions.

/* This is a multi-line comment in C */
// This is a single-line comment (C99 and later)

// Preprocessor directive
#include   // Standard Input/Output library

// Global variable declaration
int globalVar = 10;

// Function declaration (prototype)
int add(int a, int b);

// Main function - entry point of the program
int main() {
    // Local variable declaration
    int num1 = 5;
    int num2 = 7;
    int sum;
    
    // Function call
    sum = add(num1, num2);
    
    // Output statement
    printf("Sum of %d and %d is %d\n", num1, num2, sum);
    
    // Return statement
    return 0;  // Indicates successful execution
}

// Function definition
int add(int a, int b) {
    return a + b;
}

The structure above demonstrates the basic elements found in most C programs. The preprocessor directive #include tells the compiler to include the stdio.h header file, which contains necessary functions like printf(). The main() function serves as the entry point for program execution.

Data Types and Variables in C

C offers several built-in data types that allow programmers to work with different kinds of data:

// Basic data types in C
int integer_var = 10;               // Integer: typically 4 bytes (-2,147,483,648 to 2,147,483,647)
float floating_var = 3.14;          // Single-precision floating point: typically 4 bytes
double double_var = 3.14159265359;  // Double-precision floating point: typically 8 bytes
char character_var = 'A';           // Character: 1 byte (-128 to 127 or 0 to 255)

// Type modifiers
short int short_var = 100;          // Short integer: typically 2 bytes
long int long_var = 1000000L;       // Long integer: typically 4 or 8 bytes
unsigned int unsigned_var = 65535;  // Unsigned integer: only positive values

// Derived data types
int array[5] = {1, 2, 3, 4, 5};     // Array of integers
char string[] = "Hello";            // String (array of characters)

// Pointer variables
int *ptr = &integer_var;            // Pointer to an integer (stores address of integer_var)

// Type definition
typedef unsigned int uint;          // Creating a user-defined type name
uint positive_num = 500;            // Using the type definition

Variables in C must be declared before they can be used. The declaration specifies the variable's name and data type, and optionally assigns an initial value. C is a statically typed language, meaning that variable types are checked at compile time rather than runtime.

Memory Management in C

One of C's most powerful features is its ability to directly manage memory. This is done through pointers and dynamic memory allocation functions:

#include 
#include   // Required for dynamic memory allocation

int main() {
    // Pointer declaration
    int *ptr;
    int n = 5;
    
    // Dynamic memory allocation
    ptr = (int*) malloc(n * sizeof(int));  // Allocate memory for n integers
    
    // Check if memory allocation was successful
    if (ptr == NULL) {
        printf("Memory allocation failed\n");
        return 1;
    }
    
    // Using the allocated memory
    for (int i = 0; i < n; i++) {
        ptr[i] = i * 10;
        printf("ptr[%d] = %d\n", i, ptr[i]);
    }
    
    // Reallocating memory
    n = 10;  // New size
    ptr = (int*) realloc(ptr, n * sizeof(int));
    
    if (ptr == NULL) {
        printf("Memory reallocation failed\n");
        return 1;
    }
    
    // Initialize new elements
    for (int i = 5; i < n; i++) {
        ptr[i] = i * 10;
    }
    
    // Print all elements
    for (int i = 0; i < n; i++) {
        printf("After realloc: ptr[%d] = %d\n", i, ptr[i]);
    }
    
    // Free allocated memory when done
    free(ptr);
    
    // Avoid dangling pointer
    ptr = NULL;
    
    return 0;
}

Memory management in C requires careful attention from the programmer. Unlike higher-level languages with garbage collection, C programmers are responsible for allocating and freeing memory explicitly. This provides greater control but also introduces potential issues like memory leaks and dangling pointers.
